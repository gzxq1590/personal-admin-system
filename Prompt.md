# 1. AI行動規範（最上位ルール）
- 言語はすべて日本語を使用すること（コード内コメント、ドキュメント、コミットメッセージ含む）。
- 環境に対する操作（コマンド実行、ファイル作成・上書き等）を行う前には、必ず作業計画を提示し、ユーザーに「y/n」で承認を求め、「y」が返るまで一切実行しないこと。
- ユーザーの仕様を独自の判断で勝手に変更・最適化しないこと。問題発生時は独自の判断で進めず、状況と代替案をユーザーに報告して指示を仰ぐこと。
- 作業用フォルダ（`backend/` や `frontend/` 直下）にテストログやテキストファイルなどの「一時的なゴミファイル」を絶対に残さないこと。

# 2. システム全体概要とアーキテクチャ
個人利用に特化した、動作が軽量で保守性の高い事務管理システム。
対象モジュールは「ダッシュボード（サマリーと予定）」「ドキュメント情報」「タスク・工程」「経費」の4つ。

- **ディレクトリ構成**: プロジェクトのルートディレクトリ `Pj01` の直下に `backend`, `frontend`, `docs` の3フォルダを配置する。
- **バックエンド**: Python, FastAPI, SQLAlchemy, SQLite
- **フロントエンド**: Node.js, Vue 3 (Vite), Vanilla CSS
  - Tailwind等の外部CSSは非推奨。自作のCSSで清潔感と視認性を重視したシンプルかつプレミアムなデザインを構築すること。
  - Viteのプロキシ設定 (`server.proxy`) を利用し、`/api` へのリクエストをバックエンド (`http://localhost:8000`) へ転送する構成にすること。

# 3. 機能要件詳細 (CRUD実装)
すべてのデータはデータベースに永続化し、VueコンポーネントからAPI経由でCRUD操作ができること。

1. **ドキュメント管理**: ファイル本体ではなく「管理情報」を登録。項目名「ドキュメント名」「登録日」。
2. **タスク管理**: 項目「タイトル」「詳細」「ステータス」。カンバンボード風に表示し、ステータスを動的変更可能とする。
3. **経費管理**: 項目「日付」「品目」「カテゴリ」「金額」。UI上の金額はカンマ区切り（例: ¥5,000）で表示。
4. **ダッシュボード**: 不要なシステム通知枠は作らず要素を広く配置。「未完了タスク数」「ドキュメント総数」「**当月（今月）のみ**の経費合計金額」のサマリーと、直近の予定リストを表示。

# 4. 品質保証 (TDDとBDDの両立) とカバレッジ要件
品質を最重要視し、以下のアプローチでテストを実装し、**C1カバレッジ（分岐網羅率）100%を達成すること**。

1. **ユニットテスト (TDD / pytest)**
   - 全CRUDエンドポイントおよびダッシュボード集計ロジックに対し、正常系・異常系（404エラー等）のテストを実装する。
   - `pytest-cov` 等を用いてC1カバレッジを測定し、メインロジック（[main.py](cci:7://file:///c:/Users/root/Documents/sample/Pj01/backend/app/main.py:0:0-0:0)等）のカバレッジ100%を必須とする。
2. **振る舞いテスト (BDD / behave)**
   - 自然言語（日本語のGherkin構文）でユーザーシナリオ ([.feature](cci:7://file:///c:/Users/root/Documents/sample/Pj01/backend/features/expenses.feature:0:0-0:0) ファイル) を書き、Pythonによるステップ定義 ([steps.py](cci:7://file:///c:/Users/root/Documents/sample/Pj01/backend/features/steps/expenses_steps.py:0:0-0:0)) を実装して自動化テストを通すこと。
   - 新規作成・更新・削除の正常系（ハッピーパス）に加え、入力値不正（422）や存在しないIDへのアクセス（404）といった異常系（Negative Tests）のシナリオも網羅すること。

# 5. ドキュメントおよびエビデンス管理要件
作業ファイルが乱雑にならないよう、必ず以下のディレクトリ構成とファイルを作成・運用すること。

1. **ドキュメント類 (`Pj01/docs/` 配下)**
   - `startup_guide.md` (起動・停止と、Viteポート競合時の対処法)
   - `operation_manual.md` (操作手順書)
   - `test/test_report.md` (試験一覧), `test/bug_report.md` (バグ収束曲線をmermaidで図示)
2. **テスト・エビデンス類 (`Pj01/docs/evidence/` 配下へ厳格に配置)**
   - **絶対に `Pj01/backend/` 等のソースコード領域の中にログを放置してはいけない**。
   - ターミナルへ出力したテスト実行ログ（`.txt`）や、`pytest-cov` で出力したHTML形式のカバレッジレポートフォルダ (`htmlcov/`) は、すべてこの `evidence/` ディレクトリ配下に集約して保存すること。
